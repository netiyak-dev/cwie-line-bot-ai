# AGSP Skill Framework V2
## แบบประเมินทักษะตนเองสำหรับนักศึกษาหลักสูตรวิทยาศาสตร์การเกษตร
### อ้างอิง: Program Learning Outcomes (PLOs) หลักสูตรวิทยาศาสตร์การเกษตร มหาวิทยาลัยมหิดล

---

## 1. วัตถุประสงค์ของแบบประเมิน

แบบประเมินนี้จัดทำขึ้นเพื่อให้นักศึกษาสามารถประเมินตนเองด้าน **Hard Skill** และ **Soft Skill** ที่จำเป็นต่อการเรียน การทำโครงงานวิจัย การฝึกงาน และการพัฒนาตนเองในสาขาวิทยาศาสตร์การเกษตร โดยผลการประเมินใช้เพื่อสะท้อนจุดแข็ง จุดที่ควรพัฒนา และแนวทางพัฒนารายบุคคล

แบบประเมินนี้ **ไม่มีผลต่อคะแนนรายวิชา** และควรตอบตามประสบการณ์จริงของตนเองมากที่สุด เพื่อให้ระบบหรืออาจารย์ที่ปรึกษาสามารถให้คำแนะนำได้เหมาะสม

# 5. คำถามประเมินตนเอง

## HS1 — ทักษะภาคสนามและการผลิตพืช/สัตว์
**อ้างอิง: PLO1.3, PLO3.1, PLO3.2**

### Q1. เมื่อต้องทำงานเกษตรภาคสนาม นักศึกษาสามารถเลือกวิธีการ เครื่องมือ หรืออุปกรณ์ที่เหมาะสมกับงานได้มากน้อยเพียงใด?

- 🔴 ยังไม่แน่ใจว่าควรเลือกวิธีการหรือเครื่องมือใด ต้องให้ผู้อื่นแนะนำ
- 🟡 เลือกได้ในงานพื้นฐาน แต่ยังต้องปรึกษาอาจารย์ รุ่นพี่ หรือผู้มีประสบการณ์
- 🟢 เลือกวิธีการและเครื่องมือได้เหมาะสมกับงานส่วนใหญ่
- ⭐ เลือกได้อย่างมั่นใจ อธิบายเหตุผลได้ และแนะนำผู้อื่นได้

### Q2. เมื่อต้องปฏิบัติงานเกษตรภาคสนาม เช่น การปลูก ดูแล เก็บข้อมูล หรือเก็บเกี่ยว นักศึกษาสามารถทำตามหลักวิชาการและมาตรฐานได้มากน้อยเพียงใด?

- 🔴 ยังทำได้ไม่ถูกต้องหรือไม่ครบขั้นตอน
- 🟡 ทำได้บางขั้นตอน แต่ยังต้องการการดูแลหรือการตรวจสอบ
- 🟢 ทำได้ถูกต้องตามขั้นตอนและหลักวิชาการในสถานการณ์ส่วนใหญ่
- ⭐ ทำได้ดี ปรับวิธีให้เหมาะกับสถานการณ์จริง และช่วยตรวจสอบงานของผู้อื่นได้

## HS2 — ทักษะห้องปฏิบัติการและความปลอดภัย
**อ้างอิง: PLO3.1, PLO3.2**

### Q3. เมื่อต้องใช้เครื่องมือหรืออุปกรณ์ในห้องปฏิบัติการ นักศึกษาสามารถใช้งานได้อย่างถูกต้องและเหมาะสมมากน้อยเพียงใด?

- 🔴 ยังไม่มั่นใจในการใช้เครื่องมือพื้นฐาน และต้องมีผู้ช่วยใกล้ชิด
- 🟡 ใช้เครื่องมือพื้นฐานได้ แต่เครื่องมือที่ซับซ้อนยังต้องมีผู้ช่วยแนะนำ
- 🟢 ใช้เครื่องมือที่เกี่ยวข้องกับงานปฏิบัติการได้ถูกต้องในสถานการณ์ส่วนใหญ่
- ⭐ ใช้เครื่องมือได้คล่อง รวมถึงเครื่องมือเฉพาะทาง และสามารถช่วยแนะนำผู้อื่นได้

### Q4. เมื่อต้องทำงานในห้องปฏิบัติการ นักศึกษาปฏิบัติตามมาตรฐานความปลอดภัยอย่างสม่ำเสมอมากน้อยเพียงใด?

- 🔴 ยังต้องให้ผู้อื่นเตือนเรื่องความปลอดภัยอยู่บ่อยครั้ง
- 🟡 รู้กฎความปลอดภัย แต่บางครั้งยังลืมปฏิบัติหรือปฏิบัติไม่ครบ
- 🟢 ปฏิบัติตามกฎความปลอดภัยได้อย่างสม่ำเสมอ
- ⭐ ปฏิบัติเป็นนิสัย ช่วยเตือนผู้อื่น และมีส่วนร่วมสร้างวัฒนธรรมความปลอดภัย

## HS3 — การวิจัยและการจัดการข้อมูล
**อ้างอิง: PLO2.1, PLO2.3, PLO2.4**

### Q5. เมื่อต้องทำโครงงานวิจัย นักศึกษาสามารถออกแบบวิธีเก็บข้อมูล ตัวแปร และขั้นตอนการทดลองให้สอดคล้องกับคำถามวิจัยได้มากน้อยเพียงใด?

- 🔴 ยังไม่รู้ว่าจะเริ่มออกแบบการเก็บข้อมูลหรือกำหนดตัวแปรอย่างไร
- 🟡 ออกแบบได้บางส่วน แต่ยังมีข้อผิดพลาดหรือต้องให้ผู้อื่นช่วยตรวจ
- 🟢 ออกแบบการเก็บข้อมูลและตัวแปรได้เหมาะสมกับโจทย์วิจัยส่วนใหญ่
- ⭐ ออกแบบได้ครอบคลุม รัดกุม คำนึงถึงความน่าเชื่อถือของข้อมูล และอธิบายเหตุผลได้

### Q6. เมื่อต้องวิเคราะห์ข้อมูลและสรุปผลการวิจัย นักศึกษาสามารถเลือกวิธีวิเคราะห์ ตีความผล และสรุปตามมาตรฐานวิชาการได้มากน้อยเพียงใด?

- 🔴 ยังต้องให้ผู้อื่นช่วยเลือกวิธีวิเคราะห์และอธิบายผล
- 🟡 วิเคราะห์ข้อมูลพื้นฐานได้ แต่ยังต้องการคำแนะนำในการตีความหรือเขียนสรุป
- 🟢 เลือกวิธีวิเคราะห์และสรุปผลได้เหมาะสมกับโจทย์วิจัย
- ⭐ วิเคราะห์ ตีความ ตรวจสอบข้อจำกัด และให้ข้อเสนอแนะเชิงวิชาการได้

## HS4 — เทคโนโลยีสารสนเทศ เกษตรสมัยใหม่ และ AI Literacy
**อ้างอิง: PLO6.1, PLO6.2**

### Q7. เมื่อต้องค้นหาข้อมูลทางวิชาการ นักศึกษาสามารถค้นหา คัดกรอง และประเมินความน่าเชื่อถือของแหล่งข้อมูลดิจิทัลได้มากน้อยเพียงใด?

- 🔴 ยังไม่รู้ว่าจะค้นหาข้อมูลวิชาการจากแหล่งใด
- 🟡 ค้นหาได้บ้าง แต่ยังประเมินความน่าเชื่อถือของข้อมูลได้ไม่มั่นใจ
- 🟢 ค้นหาและคัดกรองข้อมูลจากแหล่งที่น่าเชื่อถือได้
- ⭐ ใช้ฐานข้อมูลวิชาการเป็นประจำ ประเมินคุณภาพแหล่งข้อมูลได้ และแนะนำผู้อื่นได้

### Q8. นักศึกษาสามารถใช้เครื่องมือดิจิทัล เช่น Excel, Google Sheets, แอปพลิเคชันเกษตร หรือซอฟต์แวร์จัดการข้อมูล เพื่อช่วยทำงานเกษตรหรือโครงงานได้มากน้อยเพียงใด?

- 🔴 ยังไม่ค่อยได้ใช้เครื่องมือดิจิทัลในการจัดการข้อมูลหรืองานเกษตร
- 🟡 ใช้เครื่องมือพื้นฐานได้ แต่ยังต้องการคำแนะนำในการจัดระบบข้อมูล
- 🟢 ใช้เครื่องมือดิจิทัลเพื่อจัดเก็บ วิเคราะห์ หรือนำเสนอข้อมูลงานเกษตรได้
- ⭐ ใช้เครื่องมือดิจิทัลได้คล่อง ปรับใช้กับงานจริง และช่วยออกแบบระบบข้อมูลเบื้องต้นได้

### Q9. นักศึกษาสามารถเข้าใจหรือประยุกต์ใช้เทคโนโลยีเกษตรสมัยใหม่ เช่น Sensor, IoT, Smart Farm, ระบบอัตโนมัติ หรือ AI tools ในงานเกษตรได้มากน้อยเพียงใด?

- 🔴 ยังไม่เคยใช้หรือยังไม่เข้าใจหลักการของเทคโนโลยีเหล่านี้
- 🟡 รู้จักเทคโนโลยีบางส่วน แต่ยังใช้ได้ในระดับพื้นฐาน
- 🟢 เข้าใจหลักการและสามารถนำไปใช้หรือเสนอแนวทางประยุกต์กับงานจริงได้
- ⭐ ประยุกต์ใช้เทคโนโลยีได้อย่างเหมาะสม พัฒนาต่อยอดได้ และคำนึงถึงข้อจำกัดของเทคโนโลยี

## HS5 — ฐานความรู้เกษตรศาสตร์และวิทยาศาสตร์
**อ้างอิง: PLO1.1, PLO1.3**

### Q10. นักศึกษาสามารถอธิบายหลักการพื้นฐานทางเกษตรศาสตร์ เช่น การเจริญเติบโตของพืช ดิน น้ำ ธาตุอาหาร โรคพืช หรือระบบการผลิต ได้ถูกต้องมากน้อยเพียงใด?

- 🔴 ยังอธิบายหลักการพื้นฐานได้ไม่ชัดเจน
- 🟡 อธิบายได้บางเรื่อง แต่ยังมีบางส่วนที่ไม่มั่นใจ
- 🟢 อธิบายหลักการสำคัญได้ถูกต้องในสถานการณ์ส่วนใหญ่
- ⭐ อธิบายได้ลึก เชื่อมโยงหลายประเด็น และช่วยให้ผู้อื่นเข้าใจได้

### Q11. เมื่อต้องวิเคราะห์ปัญหาทางการเกษตร นักศึกษาสามารถเชื่อมโยงความรู้วิทยาศาสตร์พื้นฐาน เช่น ชีววิทยา เคมี ฟิสิกส์ หรือสิ่งแวดล้อม เข้ากับปัญหาจริงได้มากน้อยเพียงใด?

- 🔴 ยังเชื่อมโยงความรู้พื้นฐานกับปัญหาเกษตรได้ไม่ชัดเจน
- 🟡 เชื่อมโยงได้ในบางเรื่อง แต่ยังต้องมีตัวอย่างหรือคำแนะนำ
- 🟢 เชื่อมโยงความรู้พื้นฐานกับปัญหาทางการเกษตรได้ดี
- ⭐ เชื่อมโยงได้เชิงลึก อธิบายเหตุและผลได้ และเสนอแนวทางแก้ไขได้

## SS1 — การคิดวิเคราะห์และแก้ปัญหา
**อ้างอิง: PLO1.1, PLO1.2, PLO1.3**

### Q12. เมื่อพบปัญหาทางการเกษตร นักศึกษาสามารถวิเคราะห์สาเหตุและวางแผนแก้ไขอย่างเป็นระบบได้มากน้อยเพียงใด?

- 🔴 มักไม่รู้ว่าจะเริ่มวิเคราะห์ปัญหาจากจุดใด
- 🟡 วิเคราะห์สาเหตุได้บางส่วน แต่แผนแก้ไขยังไม่ครอบคลุม
- 🟢 วิเคราะห์สาเหตุและวางแผนแก้ไขได้อย่างมีเหตุผล

# 7. การคำนวณผลสำหรับ Developer

## 7.1 Scoring Logic ภายในระบบ

```text
ระดับ 1 = 25 คะแนน
ระดับ 2 = 50 คะแนน
ระดับ 3 = 75 คะแนน
ระดับ 4 = 100 คะแนน
```

## 7.2 การคำนวณราย Skill

```text
Skill Score = ค่าเฉลี่ยของข้อคำถามทั้งหมดภายใน Skill นั้น

ตัวอย่าง:
HS1 Score = ค่าเฉลี่ยของ Q1 และ Q2
HS4 Score = ค่าเฉลี่ยของ Q7, Q8 และ Q9
SS5 Score = ค่าเฉลี่ยของ Q20 และ Q21
```

## 7.3 การคำนวณรายกลุ่ม

```text
Hard Skill Score = ค่าเฉลี่ยของ HS1, HS2, HS3, HS4, HS5
Soft Skill Score = ค่าเฉลี่ยของ SS1, SS2, SS3, SS4, SS5, SS6
Overall Score = (Hard Skill Score + Soft Skill Score) / 2
```

## 7.4 การแปลผลราย Skill

```text
ถ้า Skill Score ≤ 50      → ควรเสริมด่วน / Priority 1
ถ้า Skill Score 51–75     → มีพื้นที่พัฒนา / Priority 2
ถ้า Skill Score > 75      → จุดแข็ง / Strength
```

## 7.5 การเลือก Recommendation

```text
1. แสดง Recommendation สำหรับ Priority 1 ทั้งหมด
2. ถ้า Priority 1 มีน้อยกว่า 5 รายการ ให้เพิ่ม Priority 2 จากคะแนนต่ำสุดขึ้นมา
3. แสดง Recommendation สูงสุด 5 รายการต่อครั้ง
4. ถ้าทุก Skill > 75 ให้แสดง Top Strengths 3 รายการ และแนะนำการต่อยอด
```

## 7.6 การแสดงผลที่แนะนำสำหรับนักศึกษา

ไม่ควรเน้น Overall Score เป็นหลัก แต่ควรแสดงผลในรูปแบบต่อไปนี้

```text
1. ภาพรวม Hard Skill
2. ภาพรวม Soft Skill
3. จุดแข็งสำคัญของนักศึกษา
4. ทักษะที่ควรพัฒนาเร่งด่วน
5. แผนพัฒนารายบุคคล 3–5 ข้อ
6. คำแนะนำถัดไปสำหรับการเรียนหรือทำโครงงาน
```

# 8. Output Format สำหรับ Chatbot

## 8.1 ตัวอย่างข้อความสรุปผล

```text
จากคำตอบของนักศึกษา จุดแข็งหลักของนักศึกษาคือ [Skill Name] ซึ่งสะท้อนว่านักศึกษามีพื้นฐานที่ดีในการ [พฤติกรรมหลัก]

ทักษะที่ควรพัฒนาเร่งด่วนคือ [Skill Name] เนื่องจากยังต้องการการฝึกฝนเพิ่มเติมในด้าน [ประเด็นเฉพาะ]

ข้อเสนอแนะเบื้องต้นคือ
1. [Recommendation 1]
2. [Recommendation 2]
3. [Recommendation 3]
```

## 8.2 หลักการตอบของ Chatbot

- ใช้ภาษาสนับสนุน ไม่ตัดสินหรือตำหนิ
- เน้นว่า skill สามารถพัฒนาได้
- หลีกเลี่ยงการบอกว่านักศึกษา "อ่อน" หรือ "ไม่เก่ง"
- แนะนำ action ที่ทำได้จริงภายใน 1–2 สัปดาห์
- หากพบหลายทักษะที่ต้องพัฒนา ให้จัดลำดับความสำคัญ ไม่แนะนำมากเกินไปในครั้งเดียว

# 9. หมายเหตุสำหรับการนำไปใช้ใน Google Form

## 9.1 รูปแบบคำถาม

- ใช้ประเภทคำถามแบบ Multiple Choice หรือ Linear Choice ที่แสดงเป็น label
- ไม่ควรแสดงตัวเลขคะแนนภายในแบบฟอร์ม
- ควรบังคับตอบทุกข้อเพื่อให้คำนวณผลได้ครบถ้วน

## 9.2 Metadata ที่ควรเก็บ

| Field | คำอธิบาย |
|-------|----------|
| Question ID | Q1–Q23 |
| Skill ID | HS1–HS5, SS1–SS6 |
| Skill Type | Hard Skill / Soft Skill |
| PLO Mapping | PLO ที่เกี่ยวข้อง |
| Internal Score | 25, 50, 75, 100 |
| Label | 🔴 / 🟡 / 🟢 / ⭐ |
| Recommendation Group | HS1–SS6 |

## 9.3 การใช้ข้อมูลเพื่อการพัฒนาหลักสูตร

ผลการประเมินสามารถใช้ในระดับหลักสูตรเพื่อ

- วิเคราะห์ skill gap ของนักศึกษาแต่ละชั้นปี
- ออกแบบกิจกรรมเสริมทักษะก่อนทำ Senior Project
- วางแผน advising รายบุคคล
- ประเมินความพร้อมก่อนฝึกงานหรือ CWIE
- สนับสนุนการปรับปรุงหลักสูตรตามแนวทาง OBE
