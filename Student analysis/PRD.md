# PRD — AGSP Skill Assessment & Recommendation System

## ภาพรวม

Feature เพิ่มเติมบน LINE Bot **AGSP** ที่มีอยู่แล้ว  
เป้าหมาย: ให้นักศึกษาวิทยาศาสตร์การเกษตร มหาวิทยาลัยมหิดล รู้ว่าตัวเองขาด Hard Skill / Soft Skill อะไร และได้รับคำแนะนำที่นำไปทำได้จริง  
ระบบติดตามผลระยะยาวเพื่อพัฒนาการเรียนการสอนของหลักสูตร

---

## ผู้ใช้งาน

| ประเภท | บทบาท |
|--------|-------|
| นักศึกษา | ทำแบบประเมิน ดูผล รับคำแนะนำ export ผลของตัวเอง |
| Admin | ดู dashboard รวม export ข้อมูลทั้งหมด จัดการระบบ |

---

## Flow หลัก — นักศึกษา (LINE Bot)

### Step 1: เริ่มต้น
- นักศึกษา: พิมพ์ keyword เช่น "ประเมิน" หรือ "skill" หรือกดเมนูใน LINE
- AGSP: ส่ง welcome message พร้อม **PDPA consent card** (Flex Message)
  - แสดงข้อความว่าเก็บข้อมูลอะไร ใช้ทำอะไร ลบได้เมื่อไร
  - มีปุ่ม "ยอมรับ" และ "ไม่ยอมรับ"
  - ถ้าไม่ยอมรับ: ระบบหยุด แจ้งว่าไม่สามารถดำเนินการต่อได้

### Step 2: กรอกรหัสนักศึกษา
- AGSP: "กรอกรหัสนักศึกษาของน้องได้เลยนะ (8 หลัก)"
- ระบบ validate format
- ถ้าเคยทำมาแล้ว: แสดงผลเดิม พร้อมถามว่าอยากทำใหม่ไหม

### Step 3: แบบประเมิน Hard Skill
- ส่งชุดคำถามแบบ Quick Reply หรือ Flex Message
- แบ่งเป็นกลุ่มทักษะ เช่น Lab / พืชศาสตร์ / เทคโนโลยีเกษตร / วิจัยและสถิติ
- ตอบระดับ: "ยังไม่เคย" / "พอรู้" / "ทำได้" / "ทำได้ดี"
- ส่งทีละกลุ่ม ไม่ถล่มทีเดียว

### Step 4: แบบประเมิน Soft Skill
- ต่อจาก Hard Skill ทันที หรือแบ่งเซสชันได้ (ถ้าออกไปกลางทาง ให้ resume ได้)
- ตอบระดับ: "ไม่มั่นใจ" / "พัฒนาได้" / "ดีพอ" / "เป็นจุดแข็ง"

### Step 5: แสดงผล
- Flex Message แสดง:
  - **จุดแข็ง** (Top 3 ทักษะ)
  - **จุดที่ควรเสริม** (Top 3 ที่ขาดมากสุด)
  - Progress bar / visual level (ไม่ใช่คะแนนดิบ)
- ปุ่ม "ดูคำแนะนำ" / "Export PDF" / "บันทึกผล"

### Step 6: คำแนะนำ (Recommendation)
- แนะนำ 3–5 รายการ เรียงตามลำดับความสำคัญ
- แต่ละรายการระบุ:
  - ทักษะที่เสริม
  - วิธีทำ (เช่น ลงวิชาเลือก / เข้า workshop / ทำโปรเจกต์)
  - เชื่อมกับ TQF / Graduate Attribute ของมหิดล
- ปุ่ม "รับการแจ้งเตือน" (opt-in follow-up)

---

## Flow Follow-up — ระบบติดตาม

### การส่งการเตือน
- ส่ง push message หลังจากได้รับคำแนะนำ: **2 สัปดาห์ / 1 เดือน / 3 เดือน**
- ห้ามส่งเกิน 1 ครั้ง/สัปดาห์
- นักศึกษา opt-out ได้ทุกเมื่อ

### ข้อความ Follow-up
- "น้องได้ลองทำตามที่ AGSP แนะนำบ้างไหม?"
- ตัวเลือก: ✅ ทำแล้ว / ⏳ ยังไม่ได้ทำ / ❌ ทำไม่ได้

### ถ้า "ทำแล้ว"
- ถามว่าผลเป็นอย่างไร (free text สั้นๆ)
- บันทึก outcome

### ถ้า "ยังไม่ได้ทำ" หรือ "ทำไม่ได้"
- ถามเหตุผล (quick reply: ไม่มีเวลา / ไม่รู้จะเริ่มต้นยังไง / ทรัพยากรไม่พอ / อื่นๆ)
- บันทึก reason
- AGSP อาจแนะนำทางเลือกที่ง่ายกว่า

## Admin Dashboard (Web App แยก)

### หน้า Overview
- จำนวนนักศึกษาที่ทำแบบประเมิน (รวม / รายเดือน)
- กราฟ: ทักษะที่ขาดมากที่สุด (Hard / Soft แยก)
- อัตราการ follow-up response

### หน้า Cohort Analysis
- กรองตามปีการศึกษา / ชั้นปี (ถ้ามีข้อมูล)
- เปรียบเทียบผลก่อน-หลัง follow-up

### หน้า Export
- Export CSV: ข้อมูลทั้งหมด (anonymized / identified)
- กรองช่วงวันที่ได้

### หน้า Settings
- จัดการ skill framework (เพิ่ม/แก้ไข hard skill และ soft skill)
- ตั้งเวลาส่ง follow-up
- จัดการ PDPA / ลบข้อมูลนักศึกษา

---

## นักศึกษา Export

- กด "Export PDF" ใน LINE
- ได้ไฟล์ PDF 1 หน้า:
  - ผลประเมิน (visual)
  - คำแนะนำ
  - วันที่ประเมิน
  - โลโก้ AGSP / มหิดล

---

## Non-functional Requirements

| ด้าน | ข้อกำหนด |
|------|---------|
| PDPA | ต้องได้ consent ก่อนเก็บข้อมูลทุกครั้ง มีช่องทาง request ลบข้อมูล |
| Privacy | ไม่เก็บข้อมูลส่วนตัวนอกจากรหัสนักศึกษา (hashed) |
| Uptime | 95% ขึ้นไป (LINE Bot ตอบสนองได้ตลอด) |
| Push Limit | ไม่เกิน 1 push/สัปดาห์/คน |
| Resume | ถ้าออกจาก LINE กลางคัน กลับมาทำต่อได้ใน 24 ชั่วโมง |

## Milestone แนะนำ

| Phase | งาน | ระยะเวลาประมาณ |
|-------|-----|--------------|
| 1 | Skill Framework Mapping (TQF + มหิดล) + ออกแบบคำถาม | 2 สัปดาห์ |
| 2 | LINE Bot: PDPA → กรอก ID → แบบประเมิน → แสดงผล | 3 สัปดาห์ |
| 3 | Recommendation Engine + Export PDF | 2 สัปดาห์ |
| 4 | Follow-up System + Push Message | 1 สัปดาห์ |
| 5 | Admin Dashboard (MVP) | 2 สัปดาห์ |
| 6 | Pilot กับนักศึกษากลุ่มเล็ก + ปรับ | 2 สัปดาห์ |
